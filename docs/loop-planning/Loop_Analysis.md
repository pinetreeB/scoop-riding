# Loop 프로젝트 분석 보고서

## 1. 프로젝트 개요

Loop는 **전동 2륜(킥보드, 전기자전거, 전기오토바이) 라이더를 위한 통합 모빌리티 플랫폼**으로, SCOOP 앱과 같은 동업자가 기획한 별도의 신규 프로젝트이다.

### 핵심 가치 제안
- 100% 호환 보장 부품 마켓플레이스
- 3초 만에 참가 가능한 그룹 라이딩
- 자동 정비 주기 관리

### 비즈니스 모델
- 부품 거래 수수료 10-15% (핵심 수익)
- 프리미엄 구독 $4.99/월
- 정비소 예약 수수료, 프리미엄 리스팅

## 2. 문서별 요약

### WHY (왜 Loop가 필요한가?)
- 4대 Pain Point: 그룹 라이딩 조직 어려움, 부품 호환성 문제($500M 낭비), 정비 주기 관리 부재, 주행 중 HUD 부재
- 글로벌 시장: 전동 2륜 $210B+, 부품 애프터마켓 $30-40B
- 손익분기점: 25,000 MAU (MVP 출시 후 6-9개월)

### WHO (누가 사용하는가?)
- 1순위: 파워 유저 (32세 개발자 "김테크", 기기 3대, 월 300km, 연 $840 기여)
- 2순위: 소셜 라이더 (28세 마케터 "박소희", 기기 1대, 월 80km, 바이럴 효과 핵심)
- 안티 페르소나: 공유 킥보드만 사용, 미소유자, 렌탈 전용
- 지역 확장: 서울 → 샌프란시스코 → 뉴욕 → 유럽

### HOW (어떻게 실행할 것인가?)
- 기술 스택: Node.js + PostgreSQL + React Native + AWS Seoul
- DB: 45개 테이블, 150+ API 엔드포인트
- 팀: 6명 (CEO, CTO, Frontend, Backend, Designer, Marketer)
- 예산: $500K Seed Round, 6개월 MVP
- 사용자 확보: 네이버 카페 50명 → 오프라인 30명 → 그룹 라이딩 20명 = 100명 시작

### API Specification v2
- Base URL: https://api.loopandloopy.com/v1
- 인증: JWT Bearer Token (Access 15분, Refresh 7일)
- 19개 카테고리, 150+ 엔드포인트
- WebSocket 4개 (그룹 채팅, 실시간 위치, 1:1 채팅, 알림)
- Stripe 결제 연동, 배송 추적 Webhook

## 3. SCOOP과의 관계 분석

### 공통점
- 동일 동업자가 기획
- 전동킥보드/2륜 라이더 대상
- 그룹 라이딩 기능 포함
- React Native 기반 모바일 앱

### 차이점
| 항목 | SCOOP | Loop |
|------|-------|------|
| 핵심 기능 | 주행 기록, AI 분석, 배지 | 부품 마켓플레이스, 정비 관리 |
| 수익 모델 | 미정 (기록 앱) | 거래 수수료 10-15% |
| 백엔드 | MySQL + GCP | PostgreSQL + AWS |
| 규모 | 소규모 (개인 프로젝트) | 스타트업 (6인 팀, $500K) |
| 현재 상태 | 개발 완료 (v0.1.8) | 기획 단계 (문서만 존재) |
| 마켓플레이스 | 없음 | 핵심 기능 |
| 결제 시스템 | 없음 | Stripe 연동 |

## 4. 강점 분석

### 4.1 시장 분석이 탄탄함
- 글로벌 시장 규모, 타겟 사용자 수, 경쟁사 분석이 구체적
- 부품 호환성 문제라는 명확한 Pain Point 발굴

### 4.2 사용자 확보 전략이 구체적
- 단계별 (100명 → 1,000명 → 5,000명) 성장 계획
- 채널별 예상 전환율까지 제시
- 오프라인 + 온라인 혼합 전략

### 4.3 API 설계가 체계적
- User Journey 기반 API 도출 (Design Thinking 접근)
- 150+ 엔드포인트가 기능별로 잘 분류
- 에러 코드, Rate Limiting, 페이지네이션 등 실무 고려

### 4.4 리스크 관리 포함
- 사용자 미확보, 판매자 미확보, 호환성 보장 실패 등 3대 리스크 대응

## 5. 우려 사항 및 리스크

### 5.1 기술적 복잡도가 매우 높음
- 45개 DB 테이블, 150+ API, WebSocket 4개, Stripe 결제, 배송 추적
- 6명 팀으로 6개월 내 완성하기에는 매우 도전적
- SCOOP 앱 개발 경험상, 실제 개발 시간은 예상의 2-3배 소요

### 5.2 마켓플레이스 Cold Start 문제
- 판매자 없으면 구매자 없고, 구매자 없으면 판매자 없음
- "드롭쉬핑" 대안이 있지만, 배송 2주+ 소요로 UX 저하
- 호환성 DB 구축 자체가 엄청난 작업량

### 5.3 수익 모델 검증 필요
- 월 GMV $40K (5,000 MAU) → 수익 $4K는 6명 인건비 $30K에 한참 부족
- 손익분기점 25,000 MAU는 한국 전동킥보드 시장에서 달성하기 어려울 수 있음
- 한국 전동킥보드 개인 소유자 수가 실제로 얼마인지 검증 필요

### 5.4 SCOOP과의 관계 불명확
- SCOOP의 주행 기록/AI 분석 기능이 Loop에도 포함되는지?
- 두 앱을 병행 개발할 리소스가 있는지?
- SCOOP을 Loop로 통합하는 것이 더 효율적이지 않은지?

### 5.5 글로벌 확장 시기상조
- 서울에서 5,000 MAU도 달성 전에 샌프란시스코 진출 계획
- 각 지역마다 다른 규제, 언어, 문화 대응 필요
- 집중과 선택이 필요

### 5.6 예산 현실성
- $500K Seed Round로 6개월 운영 → 월 $83K 소진
- 인건비 $30K + 마케팅 $16K + 인프라 $3K = 월 $49K (최소)
- 우발 비용 $67K (13.4%)는 부족할 수 있음

## 7. 종합 평가

Loop 프로젝트는 **기획 수준이 매우 높고 체계적**이다. WHY-WHO-HOW 프레임워크로 시장 분석, 타겟 정의, 실행 계획을 잘 정리했다. API 명세도 실제 개발에 바로 사용할 수 있을 정도로 상세하다.

다만, **실행 난이도가 매우 높다**. 마켓플레이스 + 소셜 + 정비 관리 + 실시간 위치 + 결제를 모두 6개월 안에 구현하는 것은 경험 많은 팀에게도 도전적이다. MVP 범위를 더 좁히는 것이 현실적이다.

SCOOP 앱과의 시너지를 고려하면, **SCOOP의 주행 기록/소셜 기능을 Loop의 기반으로 활용**하고, 마켓플레이스를 점진적으로 추가하는 전략이 더 효율적일 수 있다.
