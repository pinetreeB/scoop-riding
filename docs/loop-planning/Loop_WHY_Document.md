# Loop 프로젝트 - WHY

## 왜 Loop가 필요한가?

**작성일: 2026년 2월 13일**

---

## Executive Summary

Loop는 전동 2륜 라이더를 위한 통합 모빌리티 플랫폼입니다. 매년 라이더들은 호환되지 않는 부품을 구매하여 약 $500M을 낭비하고 있습니다. Loop는 100% 호환 보장 부품 마켓플레이스를 제공하고, 그룹 라이딩 조직의 진입 장벽을 3초로 낮추며, 정비 주기를 자동으로 관리합니다.

**핵심 가치 제안:** "당신의 전동 2륜, 직접 고치세요. Loop가 도와드립니다."

**비즈니스 모델:** 부품 거래 수수료 10-15% (예상 연 매출 $1.14M at 100K MAU)

---

## 1. 문제 정의 (Problem Statement)

### 1.1 Pain Point #1: 그룹 라이딩 조직의 어려움

#### 현재 상황
- 전동 2륜 라이더의 30-40%가 그룹 라이딩에 관심 있음
- 하지만 실제 참여율은 10-15%에 불과 (참여 전환율 25-50%)
- 나머지 15-25%는 "참여 방법을 몰라서" 포기

#### 구체적 문제

**1) 진입 장벽 (Discovery Problem)**
- SNS 단톡방은 초대받아야만 입장 가능
- 어떤 커뮤니티가 있는지조차 모름

**2) 폐쇄적 커뮤니티**
- 기존 멤버들만의 리그
- 신입은 눈치 보임

**3) 낮은 접근성**
- 카톡 알림 폭탄
- 페이스북 정보 찾기 어려움
- 디스코드 사용법 모름

**4) 정보 비대칭**
- 그룹 라이딩 일정 공유 안 됨
- 참여 방법 불명확

#### 사용자 여정 분석

**현재 (As-Is):**
1. 구글 검색으로 커뮤니티 발견 (네이버 카페, 페이스북 그룹)
2. 가입 신청 후 3일 대기
3. 승인 후 게시글 읽으며 정보 찾기 (산발적)
4. 카톡 오픈채팅 링크 발견 → 800명, 하루 1,000개 메시지
5. **결과: 3일~1주일 소요, 70% 포기율**

**Loop 사용 시 (To-Be):**
1. Loop 앱 열기 → "그룹 라이딩" 탭
2. 지도에서 근처 그룹 확인 (날짜, 거리, 난이도, 기기 종류)
3. "참가하기" 버튼 클릭 → 즉시 완료
4. **결과: 3분 완료, 99% 시간 단축**

---

### 1.2 Pain Point #2: 부품 호환성 지옥 (핵심 수익 모델)

#### 현재 상황
전동 2륜 라이더들은 정비 비용을 절감하기 위해 부품을 직접 구매하려 하지만, **50% 확률로 호환되지 않는 부품을 구매**하여 매년 약 $500M을 낭비합니다.

#### 구체적 문제

**문제의 근본 원인:** 같은 모델이라도 제조 시기, 지역, 버전에 따라 부품이 다름

**예시 시나리오 - "김철수, 배터리 교체":**
1. 정비소 견적: 20만원 → 알리익스프레스 직구 결정 (5만원)
2. "Xiaomi Mi Pro 2 battery" 검색 → 50개 결과
3. 상품 설명: "48V 12.8Ah for Xiaomi" → 구매 ($89)
4. 2주 후 배송 도착 → **커넥터가 다름 (4핀 vs 6핀)**
5. 결과: $89 날림 + 2주 시간 낭비 → 결국 정비소 ($200) = **총 $289 손실**

#### 사용자 Pain Points
- **호환성 불확실성:** 사서 안 맞으면 돈 날림
- **시간 낭비:** 배송 2주 + 반품 2주 = 한 달
- **정보 부족:** 어떤 게 내 기기에 맞는지 모름
- **숨겨진 비용:** 결국 정비소行

#### Loop의 솔루션: 100% 호환 보장 마켓플레이스

**핵심 기능:**
- 기기 상세 등록 시스템 (브랜드, 모델, 시리얼 번호, 제조년도, 지역)
- 호환성 데이터베이스 (관리자 검증 + 커뮤니티 크라우드소싱)
- 호환 부품만 표시 (호환율 100%, 95%, 90% 등급별)
- 100% 호환 보장 프로그램 (안 맞으면 Loop가 책임)

**마켓플레이스 모델:**
- Phase 1: 순수 중개 플랫폼 (수수료 10-15%)
- 판매자: 알리익스프레스 셀러, 제조사, 로컬 부품 숍
- Phase 2: 인기 부품 자체 재고 (마진 30-50%)

---

### 1.3 Pain Point #3: 정비 주기 관리 부재

#### 현재 상황
대부분의 라이더들은 **반응적(Reactive) 정비**를 합니다. 즉, 뭔가 이상해지거나 고장 나서야 정비소를 방문합니다.

#### 문제점
- 소모품 교체 시기를 까먹음 (타이어, 브레이크, 배터리)
- 주행 거리 계산이 귀찮음
- 결과: 이미 손상이 심각해진 후 정비 → 수리비 증가, 사고 위험

#### Loop의 솔루션
- **자동 추적:** 주행 거리 자동 기록, 부품별 권장 교체 주기 DB
- **맞춤형 알림:** "타이어 교체 시기입니다 (3,000km 도달)"
- **원클릭 해결:** 알림 → 부품 구매 또는 정비소 예약

**부품 마켓플레이스와 시너지:** 정비 알림이 부품 구매로 직결 → 수익 증가

---

### 1.4 Pain Point #4: 주행 중 HUD/속도계 (보조 기능)

#### 현재 상황
그룹 라이딩 중 실시간 속도, 거리, 시간 등을 확인하기 위해 스마트폰을 자주 확인해야 하나 위험합니다.

#### Loop의 솔루션
- 주행 중 HUD (Head-Up Display): 속도, 거리, 시간 등 최소 정보
- 출발지 경로 안내: 모임 장소까지 구글맵 링크 제공
- 그룹 라이딩 중에는 앞사람 따라가면 되므로 내비게이션 불필요

**우선순위:** P0 (MVP 필수) - HUD만 기본 구현

### 3.1 경쟁사 분석

| 경쟁사 | 강점 | 약점 | Loop 차별화 |
|-------|-----|-----|-----------|
| **Strava** | 강력한 소셜, 대규모 커뮤니티 | 킥보드 지원 약함, 정비 관리 없음 | 전동 2륜 전용, 부품 마켓플레이스 |
| **알리익스프레스** | 저렴한 가격 | 호환 불확실, 배송 느림 | 100% 호환 보장, 설치 가이드 |
| **제조사 앱** | 기기 연동 | 브랜드별 분산, 소셜 없음 | 모든 브랜드 통합, 커뮤니티 |

### 3.2 Loop의 독특한 가치

- **유일한 통합 솔루션:** 아무도 "소셜 + 부품 마켓플레이스 + 정비 관리"를 통합하지 않음
- **호환성 데이터 자산:** 기기 등록을 통한 크라우드소싱 데이터 (복사 불가능)
- **네트워크 효과:** 사용자 많음 → 판매자 많음 → 부품 선택 많음 → 더 많은 사용자
- **전동 2륜 전용 최적화:** Strava는 자전거, 우리는 킥보드/전동 2륜

## 4. 수익 모델 (Revenue Model)

### 4.1 핵심 수익원: 부품 마켓플레이스

**Phase 1: 순수 중개 플랫폼**
- 거래 수수료: 10-15%
- 판매자 입점비: 무료
- 결제 수수료: 3% (Stripe)

**Phase 2: 자체 재고 추가**
- 인기 부품 직접 구매 → 창고 보유
- 마진: 30-50% (vs 수수료 10%)

#### 수익 계산 (예시)

| MAU | 월 GMV | 월 수익 (10%) |
|-----|--------|-------------|
| 10,000 | $40,000 | $4,000 |
| 100,000 | $400,000 | $40,000 |
| 500,000 | $2,000,000 | $200,000 |

**가정:** 5% 구매율, 평균 주문액 $80, 수수료 10%

### 4.2 추가 수익원

## 5. 왜 지금인가? (Why Now?)

### 5.1 시장 타이밍

- **시장 성숙:** 개인 소유자 수가 임계점 도달 (5천만 명)
- **환경 의식 증가:** 탄소 중립, 친환경 교통 보조금
- **배터리 기술 발전:** 가격 40% 하락, 용량 증가
- **규제 완화:** 유럽 2020-2022 합법화, 미국 주별 진행

---

### 5.2 경쟁 공백

**왜 아무도 이걸 안 했나?**

1. 시장이 작았음 (이제 임계 질량 도달)
2. 호환성 데이터 구축 어려움 (우리는 크라우드소싱)
3. 모두 공유 킥보드(Lime, Bird)만 집중
4. 부품 마켓플레이스 아이디어 없음 (소셜만 생각)

## 8. 투자자 FAQ

### Q1. Strava가 복사하면 어떻게 할 것인가?

**A:** 부품 호환성 DB는 복사 불가능 (우리가 먼저 쌓은 데이터). Strava는 구독 모델이고 우리는 거래 수수료 모델 (다른 비즈니스). First-mover advantage로 네트워크 효과 확보.

---

### Q2. 왜 공유 킥보드 회사들이 안 했나?

**A:** 그들의 비즈니스 모델은 B2C 렌탈 (분당 과금). 개인 소유자 시장은 완전히 다른 비즈니스. 우리는 개인 소유자만 타겟.

---

### Q3. 제조사가 직접 플랫폼 만들면?

**A:** 제조사는 자기 브랜드만 지원 (경쟁사 제외). 우리는 모든 브랜드 통합이 강점. 오히려 제조사와 파트너십 가능성 높음.

---

### Q4. Cold Start Problem은?

**A:** 소셜(그룹 라이딩)로 초기 사용자 확보 → 기기 등록 유도 → 데이터 축적 후 마켓플레이스 오픈. 단계적 접근.

- **수익 모델:** 거래 수수료 10-15% (명확함)
- **경쟁 우위:** 유일한 통합 솔루션, 호환성 데이터 자산
