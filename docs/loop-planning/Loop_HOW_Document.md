# Loop 프로젝트 - HOW (최종)

## 어떻게 실행할 것인가?

**버전:** 2.0  
**작성일:** 2026-02-13  
**포함:** 데이터베이스 설계, API 명세, 마케팅 전략, 기술 스택

## 📋 Executive Summary

Loop는 6개월 MVP 개발 후 서울에서 파일럿을 시작합니다.

**핵심 지표:**
- Month 3: 2,000 MAU
- Month 6: 5,000 MAU
- Month 12: 25,000 MAU (손익분기점)

**기술 스택:**
- Backend: Node.js + PostgreSQL
- Frontend: React Native (iOS/Android)
- Infrastructure: AWS (Seoul Region)
- Real-time: WebSocket + Redis

**개발 팀:** 6명 (CEO, CTO, Frontend, Backend, Designer, Marketer)

## 7. 리스크 관리

### Risk #1: 사용자 미확보

**완화 전략:**
```
베타 100명 실패 (Week 4):
→ 오프라인 이벤트 예산 2배 ($40K)
→ Door-to-Door (정비소 20곳 순회)

1,000명 미달 (Month 3):
→ 인플루언서 2명 추가 섭외
→ 페이드 광고 $5K 집행

피벗 기준:
Month 6에 2,000명 미만
→ WHO 재정의 (배달 라이더 조기 타겟)
```

---

### Risk #2: 판매자 미확보

**완화 전략:**
```
10개 판매자 미달 (Month 2):
→ 알리 셀러 접촉 100개로 확대
→ 첫 3개월 수수료 5%로 인하

부품 SKU 부족:
→ Loop가 직접 알리에서 구매 (드롭쉬핑)

최악의 경우:
→ Phase 1에서 마켓플레이스 제외
→ 소셜 + 정비 알림만 런칭
→ 사용자 확보 후 마켓플레이스 추가
```

---

### Risk #3: 호환성 보장 실패

**완화 전략:**
```
100% 환불 정책
Loop가 반품 비용 부담
대체 부품 무료 제공
정비소 설치 50% 할인 쿠폰

예산:
월 $2K "호환 실패 대응 기금"
실패율 목표: <3%
```
