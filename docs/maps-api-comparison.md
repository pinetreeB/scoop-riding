# 네이버 클라우드 Maps vs Google Maps API 비교 분석

## 요약

SCOOP 앱의 지도 시스템 전환을 검토하기 위해 네이버 클라우드 Maps API와 Google Maps Platform을 비교 분석했습니다. 한국 시장을 타겟으로 하는 앱의 경우, 네이버 지도가 비용 효율성과 한국 지역 데이터 품질 면에서 유리합니다.

---

## 4. 개발 난이도 비교

### 네이버 클라우드 Maps

**장점:**
- 한국어 문서 제공
- 한국 개발자 커뮤니티 활성화
- 네이버 계정으로 간편 가입

**단점:**
- React Native 공식 SDK 없음 (WebView 방식 필요)
- 글로벌 확장 시 별도 지도 서비스 필요
- 커뮤니티/예제 상대적으로 적음

**예상 전환 작업량:** 3-4시간 (현재 WebView 기반이므로 API만 교체)

### Google Maps

**장점:**
- react-native-maps 공식 지원
- 풍부한 문서와 예제
- 글로벌 확장 용이

**단점:**
- 비용 부담 (사용자 증가 시)
- 한국 지역 데이터 네이버 대비 부족
- API 키 관리 복잡

**예상 전환 작업량:** 4-6시간 (react-native-maps 설치 및 전체 컴포넌트 교체)

## 5. 코드 수정 범위

### 현재 구조 (OpenStreetMap + Leaflet WebView)

```
components/
  ride-map.tsx          ← WebView 기반 지도
  friend-map.tsx        ← WebView 기반 친구 위치 지도
lib/
  gps-utils.ts          ← GPS 유틸리티 (변경 불필요)
```

### 네이버 지도로 전환 시

| 파일 | 수정 내용 | 예상 시간 |
|-----|----------|----------|
| `ride-map.tsx` | Leaflet → 네이버 지도 JS API | 2시간 |
| `friend-map.tsx` | 동일 | 1시간 |
| `app.config.ts` | 네이버 API 키 설정 | 10분 |
| `.env` | NAVER_MAP_CLIENT_ID 추가 | 5분 |
| **총 예상 시간** | - | **3-4시간** |

### Google Maps로 전환 시

| 파일 | 수정 내용 | 예상 시간 |
|-----|----------|----------|
| `ride-map.tsx` | WebView → react-native-maps | 2시간 |
| `friend-map.tsx` | 동일 | 1시간 |
| `app.config.ts` | Google Maps API 키 설정 | 10분 |
| `package.json` | react-native-maps 추가 | 10분 |
| `app/_layout.tsx` | Provider 설정 | 30분 |
| **총 예상 시간** | - | **4-6시간** |

## 6. 추천 전략

### 단기 (현재 ~ 사용자 1,000명)
**현재 OpenStreetMap 유지** 또는 **네이버 지도로 전환**
- 비용: 무료
- 이유: 개발 리소스 절약, 핵심 기능 개발에 집중

### 중기 (사용자 1,000명 ~ 10,000명)
**네이버 클라우드 Maps 사용**
- 비용: 무료 (무료 한도 내)
- 이유: 한국 사용자 대상, 비용 효율적

### 장기 (사용자 10,000명 이상, 글로벌 확장 시)
**하이브리드 전략**
- 한국: 네이버 지도
- 해외: Google Maps
- 이유: 각 지역에 최적화된 서비스 제공
